# Installation
Infermedica Component Library provides a beautiful, configurable and accessible library of basic and advanced components for faster system design and application development.
Below you can read how you can add it to your project.

<PageOutline/>

## How to install
You can create a new project or integrate Symptom Checker UI Kit into an existing application. The installation process is mostly
the same for both cases. Choose the installation that best suits to your project.

```bash
npm i @symptom-checker/ui-kit
# or
pnpm add @symptom-checker/ui-kit
```

## Usage default styles
Infermedica Component Library is a highly configurable UI components library. To configure components to your needs, you can use a lot of global
CSS variables and adjust styles to your requirements. More details you can find in <a href="/docs/getting-started-development-guide-theme-configuration--page">Theme Configuration</a> guideline.

We provide our default design if you want use it inside your project, you need to import them into the `main.js` file.

```js
import '@symptom-checker/ui-kit/src/styles/styles.scss'
```

## Webpack loaders
To load svg files in UiIcon component we use [vue-svg-loader](https://www.npmjs.com/package/vue-svg-loader).
For Vue 3 you should use next version of this loader `npm i vue-svg-loader@next`.

```js
// vue.config.js
module.exports = {
  chainWebpack: (config) => {
    const svgRule = config.module.rule('svg');

    // clear all existing loaders.
    // if you don't do this, the loader below will be appended to
    // existing loaders of the rule.
    svgRule.uses.clear();

    // add replacement loader(s)
    svgRule
      .use('vue-loader')
      .loader('vue-loader')
      .end()
      .use('vue-svg-loader')
      .loader('vue-svg-loader')
      // options is optional, below you can find our svgo settings
      .options({
        svgo: {
          plugins: [
            { removeDimensions: true },
            { removeUselessStrokeAndFill: true },
            { convertStyleToAttrs: true },
          ]
        }
      )
  },
};
```

## Usage components in SFC
You can directly import components from the Symptom Checker UI Kit and use them in a template.

### Script syntax

```html
<template>
  <UiComponentName>
</template>

<script>
import { UiComponentName } from '@symptom-checker/ui-kit'

export default {
  components: {
    UiComponentName
  }
}
</script>
```

### Script setup syntax

```html
<template>
  <UiComponentName/>
</template>

<script setup>
  import { UiComponentName } from '@symptom-checker/ui-kit'
</script>
```

### Internal components
Some components includes smaller internal components, you must import them using the full path, e.g `UiToggleButtonGroup` uses `UiToggleButton`

```html
<template>
  <UiToggleButtonGroup>
    <UiToggleButton/>
  </UiToggleButtonGroup>
</template>
<script setup>
  import { UiToggleButtonGroup } from '@symptom-checker/ui-kit'
  import UiToggleButton from '@symptom-checker/ui-kit/src/components/molecules/UiToggleButtonGroup/_internal/UiToggleButton.vue';
</script>
```
